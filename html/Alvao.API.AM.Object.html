CreateAsync(ObjectCreationModel, CancellationToken)

Asynchronously creates one or more objects based on the specified creation model.

public static Task<IEnumerable<int>> CreateAsync(ObjectCreationModel model, CancellationToken cancellationToken = default)

Parameters

model ObjectCreationModel

The creation model containing object specification details including:

  • ObjectId - The template or source object ID (tblNode.intNodeId) to copy from
  • ParentObjectId - ID of object which will be parent (tblNode.intNodeId), null for root
  • Count - Number of objects to create
  • ExcludeGeneratingPropertyKindIds - Property types for which values from numeric sequences won't be generated
cancellationToken CancellationToken

Optional cancellation token to cancel the operation

Returns

Task<IEnumerable<int>>

Collection of created object IDs (tblNode.intNodeId)

Examples

var model = new ObjectCreationModel
{
    ObjectId = 50, // Template/source object
    ParentObjectId = 30, // Parent for new objects
    Count = 5, // Create 5 copies
    ExcludeGeneratingPropertyKindIds = new List<int> { 123, 456 } // Skip auto-generation for these properties
};
var newObjectIds = await Alvao.API.AM.Object.CreateAsync(model);

Exceptions

ArgumentException

Throws when source object ID is invalid or doesn't exist

InvalidOperationException

Throws when object cannot be copied

UnauthorizedAccessException

Throws when user doesn't have rights to create object in the destination

Move(IEnumerable<int>, int, bool, bool)

Move given objects in bulk to destination object.

public static void Move(IEnumerable<int> objectIds, int destinationObjectId, bool forceCheck, bool callCustomAction = true)

Parameters

objectIds IEnumerable<int>

List of object IDs (tblNode.intNodeId)

destinationObjectId int

ID of object which will be parent (tblNode.intNodeId)

forceCheck bool

If true, object checking will not be performed (does not apply to permission checking or system objects)

callCustomAction bool

If true, object auto actions are triggered. Default value is true. This method cannot be called in running transaction (if calling it is necessary, set the callCustomAction parameter to false).

Exceptions

InvalidBulkMoveException

Throws when object in objectIds is moved to the Recycle bin and is user who is responsible for some asset or organization that has linked licenses. Can be also thrown when other conditions are not met (e.g. permissions, moving object to itself or when object is not movable).

InvalidMoveException

Throws when moving only one object in objectIds and some conditions are not met (e.g. moving object to itself).

ArgumentException

Throws when moving only one object in objectIds and it does not exist.

InvalidOperationException

Throws when at least one of the objects in objectIds is moved to the location where will be no user responsible for it or when moving PC with licenses from one organization to another organization.
